## 📋 Project Understanding

GeoConvNet is a research codebase demonstrating that four geometric domains — 1D time series, 2D images, 3D point clouds, and 3D meshes — all share the same theoretical foundation: symmetry-constrained equivariant convolution (per Bronstein et al., Geometric Deep Learning, 2021). The repo has two parallel, self-contained sub-projects: `src/classification/` and `src/segmentation/`, each with identical directory structure (`models/`, `datasets/`, `train.py`, `evaluate.py`). Target audience is mixed academic/professional: authors writing an IEEE paper while also shipping runnable PyTorch code.

## 🚨 Critical Issues

### 1. `meshcnn_classifier.py` — `MeshPool` crashes on CUDA (lines 120–125)

`remap`, `torch.arange(self.target_edges)`, and `self_idx` are created without `.device`, so they land on CPU regardless of input. When `x` is on CUDA, the line `remap[keep_idx_sorted] = torch.arange(self.target_edges)` raises `RuntimeError: Expected all tensors to be on the same device`. The segmenter's `MeshPool` correctly passes `device=x.device`.

**Fix:** Add `device=x.device` to all bare tensor constructors in `MeshPool.forward`.

---

### 2. `pointnet2_classifier.py` — FPS ratio computed over the whole batch, not per sample (line 70)

```python
idx = fps(pos, batch, ratio=self.npoint / pos.size(0))
```

`pos.size(0)` is the total number of points across all samples in the batch (e.g. 4 samples × 1024 points = 4096), so the ratio shrinks as batch size grows. For `sa3` with `npoint=1`: at batch size 4, `ratio = 1/4096 ≈ 0.00024`, causing PyG's FPS to return essentially 0 points per sample. The segmenter correctly uses fixed ratios (`0.50`, `0.25`, `0.125`) that are batch-size-independent.

**Fix:** Replace `npoint`-based runtime ratio with a fixed `ratio` stored at construction time, matching the segmenter's `SetAbstraction` design.

---

### 3. `segmentation/train.py` — `run_epoch_1d` and `run_epoch_2d` loss normalization inflates reported loss

```python
total_loss += loss.item() * y.numel()   # y.numel() = B * T
...
return total_loss / max(n, 1)           # n += y.size(0) → n is sample count B
```

Loss is accumulated weighted by `B*T` elements but divided by `B`, producing a loss value `T` times larger than what other epoch runners report. Training is unaffected but logged loss is misleading when comparing across domains.

**Fix:** Normalize by `y.numel()` (total elements).

---

### 4. Both `evaluate.py` files — `torch.load` without `weights_only`

In PyTorch ≥ 2.4, calling `torch.load(path, map_location=device)` without `weights_only=True` raises a `FutureWarning` and will become an error in future versions. The checkpoint format is a plain dict of safe types — `weights_only=True` is safe here.

**Fix:** Add `weights_only=True` to all `torch.load` calls in both evaluate scripts.

---

### 5. `classification/datasets/loaders.py` — UCR file extension mismatch (lines 62–63)

The loader expects `ECG5000_TRAIN.txt` / `ECG5000_TEST.txt` but the README instructs users to download `ECG5000_TRAIN.ts` / `ECG5000_TEST.ts` from timeseriesclassification.com. The `.ts` format has a different header structure; passing it to `np.loadtxt(path, delimiter=",")` will fail. The segmentation loader has the same assumption.

**Fix:** Clarify in the dataset README that `.ts` files must be renamed to `.txt` before use.

## ♻️ Redundancy Findings

### 1. `ResBlock2D` duplicated verbatim
Defined identically in `conv2d_classifier.py` and `conv2d_segmenter.py`. Since the sub-projects are intentionally self-contained, this is an acceptable design choice — but it means a bug in one won't be caught in the other. Flagged for awareness.

### 2. `MeshConv` / `MeshPool` logic duplicated
Between `meshcnn_classifier.py` and `meshcnn_segmenter.py` (minor variable renaming: `in_channels/out_channels` vs `in_ch/out_ch`). Bug #1 (device mismatch) exists only in the classifier because the two copies diverged.

### 3. `build_mlp` (classifier) vs `mlp` (segmenter)
Same function with different names across `pointnet2_classifier.py` and `pointnet2_segmenter.py`. Effectively identical given both always append a final activation.

## 💡 Improvement Suggestions

### 1. Add `antialias` to `T.Resize` calls in `segmentation/datasets/loaders.py` (lines 132, 136)
Newer torchvision prints a deprecation warning without it. Use `antialias=True` for image resize, `antialias=False` for mask resize (NEAREST interpolation).

### 2. Add comment clarifying SA3 point-count arithmetic in `pointnet2_segmenter.py`
With 1024 input points: SA1 → 512, SA2 → 128, SA3 → 16. The ratio `0.125` applied to SA2's 128-point output gives 16 points. The docstring says "N/64 points" — this is correct (1024/64=16) but could confuse readers into thinking it applies ratio 0.125 to the original N.

### 3. Add comment on `MeshUnpool` nearest-fill limitation in `meshcnn_segmenter.py`
The nearest-fill for collapsed edges uses index proximity, not feature similarity, and is non-differentiable. A comment noting this mirrors the simplified approach in the original MeshCNN repo would prevent confusion for readers familiar with the paper.

### 4. Make `compute_miou` NaN-handling explicit in `segmentation/train.py`
When no valid pixels/points exist for a class, `iou_list` may be empty and silently returns `0.0`. A brief comment prevents confusion when mIoU logs `0.0` unexpectedly mid-training.

## 📚 Documentation Gaps

### 1. `datasets/` folders have no README (both sub-projects)
Download instructions are buried in main READMEs. A `datasets/README.md` listing each dataset, its download URL, expected directory layout, and any preprocessing steps (renaming `.ts` → `.txt`, SHREC16/COSEG `.npz` structure) is the most frequently needed documentation for new users.

### 2. `models/` folders have no README (both sub-projects)
No explanation of which model corresponds to which domain, or how to add a new one. Given the paper's thesis, a brief per-model summary reinforces the architecture story.

### 3. `MeshPool` in classifier lacks note that it is a simplified version
The original MeshCNN paper uses a learned edge importance; the current L2-norm scoring is a fixed-criterion simplification. A one-line comment avoids confusion for readers familiar with the paper.

## ✅ What's Already Good

- **Per-domain epoch runners cleanly separated** with rationale comments — correct design for heterogeneous data formats, easy to extend.
- **All four model files have `__main__` smoke tests** with forward-pass check and parameter count — immediately runnable.
- **`compute_miou`** correctly handles `ignore_index=-1` (VOC void pixels) and skips absent classes from the mean.
- **`MeshUnpool`** in the segmenter properly stores `keep_idx` through the encoder and uses it for feature broadcasting.
- **`UCRMotifDataset._make_seg_labels`** documents the heuristic nature of pseudo-labels and flags that ground-truth labels should replace it.
- **Docstrings throughout** include tensor shapes in `Args/Returns`.
- **`DecoderStage1D.forward`** handles length rounding via `F.interpolate` — correctly avoids off-by-one mismatches from stride-2 on odd-length sequences.
